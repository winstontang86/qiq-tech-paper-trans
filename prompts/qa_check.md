# 质检规则（阻断级）

`qa_report.py` 对翻译产物执行以下检查。**任一阻断项不通过，流程终止并输出 `qa.md` 报告**；除非用户显式传入 `--force`。

从 `References` / `Bibliography` / `参考文献` 标题开始及其后所有内容会被视为排除区（例如 `Appendix`、补充材料等），不参与正文元素计数，也不应出现在最终译文中。

如果存在阻断项，流程还会生成 `fix_prompts/`，用于提示宿主平台或外部 LLM 执行器精准修复相关翻译单元。

## 阻断级（Blocker，default stop）

| 编号 | 检查项 | 阻断条件 |
|---|---|---|
| B1 | 正文段落数对齐 | 译文段落数 < 原文正文段落数的 90% |
| B2 | 正文图片保留 | `![...](...)` 的数量少于原文正文 |
| B3 | 正文表格保留 | Markdown 表格块数量少于原文正文 |
| B4 | 正文公式保留 | `$...$` / `$$...$$` / `\begin{equation}` 块数量少于原文正文 |
| B5 | 正文代码块保留 | fenced code block 数量少于原文正文 |
| B6 | 正文引用编号保留 | `[\d+]` / `(Author et al., Year)` 数量少于原文正文的 95% |
| B7 | 翻译单元长度比 | `len(zh_chars)/len(en_chars)` 不在 [0.3, 2.5]；支持 `translation_units.json`，否则回退到 `segments.json` |
| B8 | 摘要性短语 | 译文出现"综上所述 / 简而言之 / 总的来说 / 作者主要讨论了 / 简要介绍 / 下面我们"等短语，且原文未出现相同短语 |
| B9 | 正文锁定块完整性 | 最终译文仍残留 `⟦...⟧` 占位符，或正文中的 `locked_blocks.json` 原始锁定块未出现在最终译文中 |
| B10 | References 后内容排除 | `is_reference=True` 的 References / Bibliography 及其后段落仍出现在最终译文中 |

### 长度比区间说明

英译中字符比例在真实论文中波动较大，特别是公式、占位符、章节级翻译和术语首次出现会影响长度。因此实现采用较宽的 [0.3, 2.5]：

- 下限 0.3：捕获空译、明显漏译、摘要化压缩。
- 上限 2.5：捕获异常展开、重复翻译或解释性补写。

## 警告级（Warn，仅提示不阻断）

| 编号 | 检查项 |
|---|---|
| W1 | 术语一致性：保留为后续扩展项 |
| W2 | 中英文之间缺少空格：当前由 `postprocess.py` 自动规范化 |
| W3 | 中文段落混用半角标点：当前由 `postprocess.py` 自动规范化 |
| W4 | 疑似未翻译（翻译单元仍主要为英文） |

## 修复 prompt

当存在阻断项时，会生成：

- `fix_prompts/INDEX.md`：阻断项索引。
- `fix_prompts/<BLOCKER>_<TARGET>.fix.md`：针对具体翻译单元或全局问题的修复指令。

## 跳过机制

- `--force`：即使存在阻断项也继续输出最终文件，但仍生成 QA 报告与修复 prompt。
- `--skip-checks B7,B8`：跳过特定检查项（逗号分隔）。
